# FamFi 家計簿MVP

夫婦の共通支出をゆるく管理するための家計簿アプリです。このブランチでは、まず日々使えることを優先して、支出、入金、月次サマリに絞っています。

## いま使えること

- 支出の登録、編集、削除、一覧表示
- 入金の登録、編集、削除、一覧表示
- 計上年月による月切り替え
- 支出フィルタ（カテゴリ、支払者、財源）
- 月次サマリ
  - 支出合計
  - 共通支出（楽天、支払者立替）
  - 実費支出
  - 入金合計
  - 差分（共通）
  - 共通プール累計
- 入金 vs 共通支出の簡易棒グラフ

MVPではブラウザのlocalStorageに保存します。まず自分で使って運用感を確かめるための状態です。

## まだやっていないこと

- Supabaseへの実データ保存
- Supabase Authによる家族限定ログイン
- RLSの本番有効化
- カテゴリ、サブカテゴリのマスタ管理
- 初期残高の画面設定
- CSVインポート/エクスポート

## 起動

```bash
npm install
npm run dev
```

ブラウザで `http://localhost:3000` を開きます。

## Supabase/Prismaの準備

このブランチには、将来PostgreSQLへ移すための `prisma/schema.prisma` と初回migrationを置いています。

1. Supabaseでプロジェクトを作る
2. `.env.example` を参考に `.env.local` を作る
3. Supabaseの接続文字列とanon keyを入れる
4. Prismaを生成、migrationする

```bash
npm run db:generate
npm run db:migrate
```

Supabase Authを導入するときは、`expenses.user_id` と `deposits.user_id` に `auth.users.id` を入れます。migration内にRLS policyの雛形コメントを残しています。

## 集計ルール

- 月次サマリは支出の `計上年月` で集計します
- 支出合計は財源を問わず合計します
- 共通支出は `楽天` と `支払者立替` を合計します
- 実費支出は `実費` だけを合計します
- 差分（共通）は `入金合計 - 共通支出` です
- 共通プール累計は月ごとの差分（共通）を初期値0で累積します

## 開発メモ

既存プロジェクトには家計簿以外の画面もありましたが、このMVPブランチではトップページに家計簿機能を集約しています。サブルートはトップページへの案内ページにしています。
